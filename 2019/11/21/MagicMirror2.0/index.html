MagicMirror系列教程(二):前期准备
Created 2019-11-21| Updated 2019-11-29|MagicMirror
Post View:

再开始之前请浏览教程首页，以便访问常用链接以及总目录。

树莓派系统烧录

在官网下载最新系统镜像，放入一个全英文路径。不要下载lite版本，推荐使用torrent下载，可以断点续传，比网页下载快。


下载软件SD Association，用来格式化你的SD卡，软件放在百度网盘里。

插上读卡器，打开软件选择你要格式化的SD卡，选择Format

提示SD卡内有数据，是否要继续，选择“是”

很快就出现格式化成功的界面：

下载软件Win32 Disk Imager，用来把系统烧录进SD卡，软件放在百度网盘

打开软件，选择你刚刚下载的系统镜像以及SD卡，点击write

点击yes

然后等待大约5-10分钟，系统就烧录好啦，这时候就可以当做树莓派的系统盘了


系统烧录好之后，别急着插上树莓派，先在目录下面新建一个空文件，命名为ssh（下面ssh链接的时候会用到）注意，是空文件，没有任何的后缀名

树莓派插上SD卡，连接好显示器，拔掉鼠标等外设（为了第一次启动不出啥问题）接通电源，等待一会儿。
第一次开机会弹出这样的框子，按照提示进行初始化设置，国家选择China，设置系统密码，是否连接网络等等。系统会提示是否需要立即更新，时间充裕的话可以更新下，也可以跳过，下面有步骤会更新，最后完成就ok。

SSH连接树莓派

这里介绍SSH连接是因为：
1.以后你对pi的操作过程中，很多时候会用到SSH连接。
2.有的大段代码手敲不易，可以直接复制粘贴

树莓派端

Ctrl+Alt+T打开树莓派终端，在其中以此输入：sudo raspi-config回车，出现如下选择，选5.interfacing options

选择ssh

问是否开启SSH，选择yes

显示ssh服务已经打开了：

然后重启树莓派sudo reboot

PC端

以下两种方法均可，选其一则行。优先选择第二种。有屏幕的推荐树莓派直连路由器，比较方便，第一种方法适用于树莓派端已经开启ssh但无屏幕无法设置无线网络连接的情况。

1.用网线直连树莓派和PC

接通好网线，打开电脑的控制面板，选择网络和Internet，网络和共享中心，更改适配器设置，选择连接的无线网，右击属性，点击“共享”，选择本地共享无线网。



连接上树莓派之后，Win + R打开运行，输入cmd回车

在cmd中输入arp -a回车

192.168.137.***就是你树莓派的IP地址
如：我的是：

2.树莓派wifi或者网线连接路由器

进路由器查看自己树莓派的IP地址（路由器的进入方法自行百度，每个路由器进入方式不一样）
如，我的路由器网关地址为192.168.123.1，进入之后找到树莓派的IP地址

windows用putty连接树莓派

现在万事俱备，只差登录了。下载软件putty并打开（软件已上传至百度网盘）

输入你所查询到的树莓派的IP地址，端口选择22，连接类型SSH，打开

打开后如图，树莓派默认用户名pi，密码raspberry，开机设置时候改过密码的用自己密码（输入密码的时候密码不显示，输完回车就行）

输入完账户密码，出现下图所示就算是登录成功了，以后再终端输入的指令，就可以在这个对话框下进行了

树莓派换源

树莓派换源之后更新、安装软件等都会变快很多，我们这里换成国内的源。在此过程中，我们会被要求以root 用户来进行更新，所以我们先创建root 用户
sudo passwd root
之后会被要求输入两次相同的密码，输入完后就成功创建了root 用户了 ，但现在先不急着切换到root 用户，我们先以目前的pi 用户操作
sudo nano /etc/apt/sources.list
将原本的源全都用“#”注释掉，最后一行加入阿里源：
deb http://mirrors.aliyun.com/raspbian/raspbian/ stretch main contrib non-free rpi
Ctrl+O回车保存，Ctrl+X退出编辑器，然后再输入
sudo nano /etc/apt/sources.list.d/raspi.list
将原本源注释，加上中科大源：
deb http://mirrors.ustc.edu.cn/archive.raspberrypi.org/debian/ stretch main ui
Ctrl+o 回车保存，Ctrl+X退出编辑器。
然后切换到root 用户。
su root
最后，运行更新命令，耐心等待。
sudo apt-get update && apt-get upgrade
退出root账户：终端输入exit
重启sudo reboot
