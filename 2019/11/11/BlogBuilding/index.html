本次博客搭建心得
Created 2019-11-11| Updated 2019-11-29|博客搭建
Post View:
  心血来潮想搭建一个博客，平台来来回回选了好几个，多多少少都有点广告影响阅读，寻觅过程中知道了GitHub page这个神器，于是开始了填坑之旅，写个文章记录下心得。

1.GitHub page的使用方法

  1.在Github上面新建一个仓库
首先你得有一个GitHub账户，注册过程就自行学习了，然后点击“new”新建一个仓库

  2.仓库的名字设为XXX.github.io,”XXX”为你的用户名，如我的为Season000.github.io

  3.至此为止，GitHub page搭建就完成了，已经可以在上面部署静态网页了，例如自己写一个静态的html，域名访问
yourname.github.io/(yourname为你的用户名)便可以访问你的网页了。

  对没错，在GitHub上面搭建自己的静态网页就这么简单，只要运用git相关知识，将仓库克隆到本地，在本地部署完自己的页面之后，在push到GitHub，就可以随时随地访问自己搭建的网站啦。难点不在这里，是下面在本地预览效果的步骤。

选择jekyll还是hexo

  GitHub page只支持静态网页，对于我这种对前端只有初步了解的小白而言，从零开始搭建自己的博客是几乎不可能的，于是乎就选择使用别人的模板，当下主流的使用的框架有jekyll和hexo两种，两种我都尝试过了。
我第一次选择了jekyll，至于为什么我也不知道，两个框架都不了解，官方建议是jekyll，就选择了前者，从此在采坑的路上一去不返。。。只能说在windows上安装jekyll要有良好的心理素质，会遇到一些莫名其妙的问题。
随后我试了一下hexo，只需要安装一个node.js，相比之下友好太多，所以新手还是建议使用hexo，能节约很多踩坑的时间。两种方法我都会简单介绍下在windows下的安装。

windows下jekyll的安装

  因为GitHub page支持jekyll，所以只要部署上去就行，这里将本地代码部署到github上的方法暂不介绍，自行学习git基础知识，我们所要做的，就是在本地安装jekyll，来预览我们的博客效果，当效果符合自己的要求的时候，只要推到你的仓库，就可以访问了。

  jekyll是一个简单的免费的Blog生成工具，Linux系统下的安装较简单，这里主要介绍windows下jekyll的安装心得

  网上教程非常多，我试了好多，可谓遍地是坑，你总能遇到新的博主没写的问题，所以在这里建议的是，直接按照jekyll官网安装步骤的来！！！不要直接按照网上各种博客的教程，可以遇到问题再去搜索解决。

  在安装jekyll之前，ruby，rubygems，gcc，make的环境一定要装好，否则出来的报错简直不忍直视

  当所有的环境都安装好的时候，打开cmd，进入到你博客的文件夹将执行jekyll new myblog，在进入所创建的文件夹cd myblog，构建网站启动web服务bundle exec jekyll serve，这时候在浏览器打开127.0.0.1:4000就能看到一个初始自带的jekyll主题了
